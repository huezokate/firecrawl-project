## Context

Use the Firecrawl API to scrape README content for each repo found by the GitHub search. Firecrawl gives richer, cleaner markdown than raw GitHub API responses, which improves Claude categorization quality. Scrape the GitHub repo page (or raw README URL) and extract the first ~1000 chars for categorization context.

## Acceptance Criteria

- `backend/firecrawl_client.py` with async function `scrape_readme(repo_url: str) -> str`
- Uses `FIRECRAWL_API_KEY` from env
- Fetches and returns cleaned markdown content from the repo's GitHub page
- Gracefully handles failures (timeouts, 404s) — returns empty string, does not crash
- Truncates content to 1500 chars max before returning (cost/latency control)
- Concurrent scraping with `asyncio.gather` with a concurrency cap (max 5 at once)
