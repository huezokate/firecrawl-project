## Context

Wire up the Claude categorization pipeline and expose the full FastAPI endpoint. Claude reads each repo's description + README snippet and assigns one category. The FastAPI endpoint orchestrates: GitHub search → Firecrawl scrape → Claude categorize → return JSON. This is the single endpoint the frontend calls.

## Acceptance Criteria

- `backend/claude_client.py` with `categorize_repo(name, description, readme_snippet) -> str`
- Uses model `claude-sonnet-4-20250514` with `ANTHROPIC_API_KEY`
- Categories: RAG Pipeline, AI Agent, Lead Gen, Content Scraper, Research Automation, Dev Tool, Other
- Returns one of the exact category strings above
- `backend/main.py` FastAPI app with `GET /api/repos` endpoint
- Endpoint returns JSON: `{ repos: [...], categories: { "RAG Pipeline": N, ... } }`
- Each repo object: `{ name, description, stars, language, url, category, readme_snippet }`
- CORS configured to allow frontend origin
- Results cached in-memory for 1 hour (avoid re-hitting APIs on every page load)
- `POST /api/refresh` endpoint to bust the cache manually
